## 2026-04-26（昨日）

### SW=0005 卡片半初始化状态修复

**问题**：扫卡时 INS=0x42（derive）返回 SW=0005，卡片无法派生地址。

**根因分析（三阶段调试）**：

| 阶段 | 发现 | 结论 |
|------|------|------|
| 第一阶段 | tag 0xA1（密钥材料）全零，但 hasKeyPair 标志位=1 | 卡片处于半初始化状态，需重新调用 INS=0x41 |
| 第二阶段 | INS=0x41 执行后，旧 AES session key 失效导致 INS=0x42 SW=0005 | INS=0x41 修改了卡片 SELECT 响应，需重新 SELECT 刷新会话密钥 |
| 第三阶段 | reselect 后 getStatus 解密正常（AES key 正确），但 tag 0xA1 仍全零 | 卡片 flash 物理损坏，无法持久化密钥材料，属于硬件故障 |

**修改内容**：

`ChipCoreBlockchainApi.kt`：
- `HdWalletCardClient`：`aesKey` 改为 `var`（可更新），新增 `tagIdBytes` 字段存储 tag ID
- 新增 `reselect(aid: ByteArray)` 方法：重新 SELECT applet，用新 SELECT 响应更新 `aesKey`
- `ensureKeyAndDerive`：调用 INS=0x41 后，若 `masterKeyFlagSet=true`，执行 reselect → getStatus 验证新 session key
- 若 reselect 后 `hasKeyPair` 仍为 false，抛出 `card-hardware-error` 快速失败，不再继续到必然失败的 derive 步骤

`scan_util.dart`：
- `_fromException` 新增 `card-hardware-error` 错误码处理，UI 显示"卡片密钥存储异常，请联系设备供应商更换卡片"

## 2026-04-28（昨日）

### 1. DOGE 签名 "Script was NOT verified successfully" 修复

**根因**：NFC 卡片签名返回原始 64 字节 `r[32] || s[32]`（首字节非 `0x30`，例如 `0x56`），而旧 `normalizeSignatureDer` 只处理已是 DER 格式的签名，对非 `0x30` 首字节原样返回，导致写入交易的签名格式错误，脚本验证失败。

**修改内容**（`BlockchainEncoders.kt`）：
- `normalizeSignatureDer`：检测首字节非 `0x30` + 长度恰好为 64 时，直接将 `r[0..31]` / `s[32..63]` 解析为 `BigInteger` 并调用 `encodeDer` 转换为标准 DER
- `encodeDer`：改用 `BigInteger.toByteArray()`，当最高位为 1 时自动补 `0x00`，避免负数误解

### 2. DOGE API 首次迁移（BlockCypher → Blockchair）

BlockCypher 余额/UTXO 接口触发 429 频率限制，切换到 Blockchair：
- 余额：`api.blockchair.com/dogecoin/dashboards/address/{addr}` → `data[addr].address.balance`（satoshi）
- UTXO：同上 → `data[addr].utxo[]`（`transaction_hash/index/value`）
- 广播：`api.blockchair.com/dogecoin/push/transaction`，POST form `data=<hex>`

### 3. Kotlin 编译错误修复

**问题**：`fetchDogeUtxos` 中 `parseRefs` 局部函数使用表达式体 `= (...)` 同时内含 `return@parseRefs`，Kotlin 不允许在表达式体中使用非局部 return。

**修改**：`parseRefs` 改为块体 `{ ... }`，`return@parseRefs emptyList()` 改为 `return emptyList()`，编译通过。

## 2026-04-29（昨日）

### Android NFC Bug 修复

**Bug 1：`reselect()` 静默吞异常，导致级联 transceive 失败，Flutter 收到 null/null**

**根因**：原 `reselect()` 内 `catch (e: Exception)` 只打日志不重抛，导致异常丢失，后续 AES 解密/命令继续在已断开的 tag 上执行，每条命令都抛 `IOException("transceive failed")`，最终返回 `null, null`。

**修改**（`NfcCardSession.kt`，`reselect()`）：
```kotlin
catch (e: Exception) {
  Log.w(TAG, "reselect failed", e)
  throw e   // 重新抛出，中断后续流程
}
```
同时：SELECT 响应 SW ≠ 9000 时也主动抛 `IOException("reselect SW: ${sw1.toString(16)}${sw2.toString(16)}")`。

**Bug 2：`wrapThrowable()` 未映射 transceive IOException → nfc-tag-lost**

**修改**（`NfcCardSession.kt`，`wrapThrowable()`）：新增分支：
```kotlin
is IOException -> if (msg.contains("transceive") || msg.contains("tag") || msg.contains("lost")) {
  FlutterError("nfc-tag-lost", "NFC 连接中断，请把手机背面保持贴近卡片后重试", null)
} else { ... }
```

**Android 编译验证**：`./android/gradlew :app:compileDebugKotlin` → **BUILD SUCCESSFUL** ✓

## 2026-04-30

### iOS NFC + AES 全面实现（与 Android 对齐）

#### 背景
iOS `ChipCoreBlockchainApi.swift` 原有 NFC 操作均为空实现或无加密，与 Android Kotlin 实现存在大量功能缺口。

#### 实现内容（`ChipCoreBlockchainApi.swift`）

**1. AES-128-CBC 加解密（CommonCrypto）**

移除 CryptoSwift 依赖（GitHub 不可达），改用系统内置 `CommonCrypto`：
- `aesCbcEncrypt(data:key:)` — `CCCrypt(kCCEncrypt, kCCAlgorithmAES, 0)`，IV = key 前 16 字节，手动 PKCS7 填充
- `aesCbcDecrypt(data:key:)` — 对应解密，剥离 PKCS7 尾部填充

**2. Session 密钥推导**

`selectAppletIfNeeded`：
- SELECT 响应去掉最后 2 字节（SW）+ 卡片 UID 前 4 字节 → `aesKey`
- 传递给 `beginOperation(tag:tagId:aesKey:)` → 构造 `IOSSessionChannel`

**3. IOSSessionChannel**

- `sendRaw()` — 原始 APDU，不加密
- `send(apduData:context:skipEncrypt:)` — 当 `skipEncrypt=false` 且 `aesKey` 非空时对 APDU data 做 AES-CBC 加密，对响应做解密

**4. CardStatus 扩展**

新增字段：
```swift
let pinRetry: Int    // 0 = 已锁
let pukRetry: Int    // 0 = 已过期（卡注销）
var isLock: Bool { pinRetry == 0 && pukRetry > 0 }
var isExpired: Bool { pinRetry == 0 && pukRetry == 0 }
```
`getStatus()` 解析 TLV tag `0x9C`（pinRetry）/ `0x9D`（pukRetry）。

**5. HdWalletCardClient 完整实现**

| 方法 | 说明 |
|------|------|
| `getStatus()` | `skipEncrypt: true`，解析 pinRetry/pukRetry |
| `reselect(aid:completion:)` | 重新 SELECT，更新 `channel.aesKey` |
| `generateKeyPair()` | INS=0x41，`skipEncrypt: true` |
| `deriveKey()` | INS=0x42，`skipEncrypt: true` |
| `sign()` | 先 `reselect()`，再 AES 加密发送 |
| `sendCommand(_:completion:)` | 先 `reselect()`，再加密发送 |
| `writeNdef/readNdef/writeNdefAndVerify` | NDEF 操作（tag 0x9B） |
| `writeUid()` | INS=0x65，`sendRaw`（无加密） |

**6. HdWalletApdu 常量扩展**

| 类别 | 常量 |
|------|------|
| PIN | `insCreatePin=0x50`, `insUpdatePin=0x51`, `insUnlock=0x52`, `insCancelPin=0x57` |

**7. scanCardWithCommand 重写（对齐 Android）**

流程：
1. `getStatus()`（无加密）→ 检查 isLock / isExpired → 快速失败
